### 2. Favicon Branding Across All Sites

Implemented inline SVG emoji favicons as a lightweight, consistent favicon pattern. No external assets or build step required.

| Site | Favicon | Meaning |
|------|---------|---------|
| Photography website (https://photographybyjennalynn.com) | 📸 | Camera with flash — on-brand for a photographer |
| Portfolio (https://mattbenko.xyz) | 🛰️ | Satellite — networking + space theme |
| Caddy CMS Editor (Tailscale) | 🛠️ | Hammer and wrench — editing/workshop |
| Caddy File Manager (Tailscale) | 📂 | Open folder — file management |

**Why it matters:** Every Caddy-managed site now has a recognizable browser-tab icon. The inline SVG approach avoids image hosting, works on GitHub Pages, and loads instantly.

**Implementation detail:** Each favicon is a single `<link rel="icon">` with a `data:image/svg+xml` payload containing the emoji. The photography, portfolio, and CMS sites are version-controlled; the file manager is updated live on the VPS.

### 4. Caddy File Manager — Image Previews

| Before | After |
|--------|-------|
| File list showed only filename + generic icon | Image files (`jpg`, `jpeg`, `png`, `gif`, `webp`, `bmp`, `svg`) render a thumbnail next to their name |
| Downloads forced for every file | New `/api/preview/<path>` endpoint serves images inline with correct MIME type |
| No visual distinction for photo folders | Thumbnails make it easy to identify photos at a glance |

**Why it matters:** Uploading and organizing photos for the photography site is faster when you can see what each file is without downloading it.

**Sizing iteration:** Started at 28px, doubled to 56px at the operator's request for easier identification.

**Implementation detail:** Added `handle_preview()` in `/opt/caddy-services/scripts/file-server.py`, routed `/api/preview/`, added `.thumb` CSS, and updated the JavaScript file-table renderer. The service was restarted to apply changes.
