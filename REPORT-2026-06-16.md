## Summary

Today we leveled up Caddy's external capabilities by adding authenticated access to GitHub, enabling web search and scraping through Firecrawl, and completing several optional-but-useful Hermes toolset prerequisites. All changes are reversible, documented, and scoped to the `~/.hermes/` profile on the VPS.

## What Changed

### 1. GitHub Integration — Full Assistant Access

| Before | After |
|--------|-------|
| Anonymous GitHub API calls (60 req/hr limit) | Authenticated as `BenkoMatt` via `GITHUB_TOKEN` |
| No `gh` CLI | `gh` CLI v2.94.0 installed and logged in |
| `github-pr-workflow` skill present but unused | Can now list, create, merge PRs; manage issues; check CI runs; fork/clone repos |

**Why it matters:** Caddy can now act as a GitHub assistant — open pull requests, review CI status, triage issues, and manage repositories directly from the terminal without rate-limit friction.

**Configuration:**
- Added `GITHUB_TOKEN` to `~/.hermes/.env`
- Authenticated `gh` CLI
- Enabled `github-pr-workflow` skill (was already installed)
- Set a cron reminder to renew the token on **July 15, 2026** (one day before expiry)

---

### 2. Web Search & Scraping — Firecrawl Enabled

| Before | After |
|--------|-------|
| `web` toolset unavailable | `web` toolset enabled with `web.backend = firecrawl` |
| No web search capability | Can search, scrape, and extract clean markdown from web pages |

**Why it matters:** Caddy can now research live web content — check competitor sites, verify documentation, summarize articles, and gather up-to-date information that local models and repos cannot provide.

**Configuration:**
- Created Firecrawl account via onboarding flow
- Added `FIRECRAWL_API_KEY` to `~/.hermes/.env`
- Installed `firecrawl-cli@latest` globally
- Set `hermes config set web.backend firecrawl`
- Enabled `web` toolset

**Plan:** Firecrawl free tier (1,000 pages/month). If usage exceeds that, we can upgrade to a paid plan or evaluate self-hosted alternatives such as SearXNG.
